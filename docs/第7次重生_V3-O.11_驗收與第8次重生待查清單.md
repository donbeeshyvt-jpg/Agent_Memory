## 四、背景觀察指示（重啟後掛這些 Monitor）

### 4.1 來回運算時間（延遲）
- **即時 tail**：`Get-Content "<vault>\.ai\turn_timings.jsonl" -Wait -Tail 5`
- **Monitor A**（per-turn）：tail turn_timings.jsonl，解析 `step15_llm_call`(deepseek 出口/彙整) / `step4_5_llm_emotion_fallback`(gemma) / `total_ms` / `top3_slowest`
- **Monitor B**（relay）：tail relay.log，抓 `[TIMING]` recv_pre/bridge_roundtrip/reply_send + 錯誤關鍵字
- **重點觀察**：① 彙整出口 deepseek 延遲（大 input 後）② gemma 是否快速 degrade 不卡 ③ 彙整 vs 個別回的端到端時間 ④ barrage 下是否 O(1) 出口（不再 N 次 deepseek）
- **基準**：deepseek 個別 3–15s / 彙整(大input)可更久但省 N 次；gemma 0.2–2s（駐留）/ degrade ~15s；reply_send 0.2–1.5s

### 4.2 大腦記憶觀察
- **DB 成長**：每隔一段 `python _db_peek.py` 看 raw_events / episodic_memories / emotion_state / affect_states / appraisal_records / intimacy_states 成長是否合理
- **朋友卡演化**：watch `20_Audience_Graph/22_Casual_Viewers/*.md` —— 反思段是否隨互動更新、對話彙整是否壓縮、intimacy_stage 是否升

### 4.3 壓縮對話情形（重點）
- **近10句對話彙整**：朋友卡「## 近期對話彙整」段是否為**壓縮摘要**（非逐句堆積）→ 驗「寫入深、讀取淺」
- **日重整（layer3 24h）**：跑後看 `viewer_cards_daily_refined(N)` + 朋友卡反思是否昇華
- **7天總彙整（layer4 7d）**：跑後看 `viewer_cards_weekly_consolidated(N)` + 舊對話是否壓縮、卡是否沒無限長
- **重要性加權**：高 emotional_salience/intimacy 的內容是否優先保留、低分淡出
- **彙整 input 大小**：彙整時 deepseek input（owner記憶+最近對話+朋友卡彙整版×N+本批）是否在合理範圍（packet 上限 160000、deepseek context 1M）
